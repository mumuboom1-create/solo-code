# K446 PbootCMS 模板高端化改造评估

参考调性：Vargov / Lasvit / Flos / Adstyle。
当前模板：PbootCMS + `template/default/*.html` 单皮肤 + `skin/css/style.css`(60KB) + `media.css`(36KB)。

---

## 一、整体结论

**可以做，工作量可控。** 客户方案里所有栏目（首页 / 品牌 / 产品系统 / 工程案例 / 定制服务 / 关于 / 联系）现有 PbootCMS 模板的标签体系（`{pboot:nav}` / `{pboot:list}` / `{pboot:sort}` / `{pboot:content}`）100% 能撑起来，**不需要改 PHP 后台**，只动 `template/default/` 9 个 html 模板 + `skin/css/style.css`。

**关键判断：** 现有模板视觉是 2018 风格的「科技公司」蓝白配色 + WOW 滚动动效，跟客户想要的高端灯具品牌路线（黑金、衬线大字、Vargov 那种暗调氛围）差距很大，**不是改色就能升级的**，需要把首页和列表页的视觉骨架重写一次。但因为后端标签不动，重写过程对后台录入完全透明，客户后续上传图文还是用熟悉的方式。

## 四、改造工作清单（按文件粒度）

### 4.1 重写（视觉骨架级）

1. `skin/css/style.css` —— 新建一份 `style.v2.css`，沿用现有类名结构尽量复用，但全面替换配色/字号/字体/间距/动效曲线。**不要在原 60KB 文件上改**，老文件保留以防回退。
2. `template/default/head.html` —— 引入 v2 css、Google Fonts（Cormorant + Inter）、新导航 markup（玻璃态 + 中英语言切换）。
3. `template/default/index.html` —— 按效果图重排 Hero / 走马灯 / 5 大系列 / 故事 / 案例 / 能力 / 流程 / 认证 / CTA 9 个 section，全部用 `{pboot:list}` `{pboot:slide}` 拉数据。
4. `template/default/foot.html` —— 4 列 footer + 邮件订阅（接 PbootCMS message 表单）。

### 4.2 改样式 + 微调结构

5. `productlist.html` —— 顶部 banner 改成黑底大字标题 + 面包屑；列表改 3 列大图卡片，hover 显示金色 `→`。
6. `product.html` —— 详情页拆成左大图 swiper + 右产品信息（型号 / 尺寸 / 材质 / 色温 / 询盘按钮），下方 tab 已经有了，重排即可。
7. `about.html` —— 引入大字 quote 板块 + 工厂九宫格 + 资质墙。
8. `contact.html` —— 表单整体重做（圆角输入、金色按钮、必填星号、邮箱+国家+项目类型字段），地图换 Google Maps iframe。
9. `newslist.html` / `news.html` —— 改成「工程案例」语义：封面图 5:4 大图卡片 + 项目标签（酒店/别墅/商业）。

### 4.3 仅替换文案/资源

10. `danye.html` —— 单页通用模板，调字号字体即可。
11. `imageslist.html` / `downlist.html` / `joblist.html` / `search.html` —— 调样式跟着新 CSS 走，结构基本不动。

## 五、客户方案点对点回应

### 1) 页面架构搭建
「全部用模板原生功能实现，不做页面代码定制」—— 这一句字面意义上做不到，因为 PbootCMS 模板本身就是 html 文件，**栏目落地必然要改 template 里的 html**，但是**不需要改后台 PHP / 数据库**。建议跟客户对齐口径：「不动后台逻辑，只动前端皮肤模板」。

### 2) 6 大栏目
全部能用现成标签实现，详见上面映射表。「静态询价按钮」= 跳到联系页锚点 `#message`，模板里 `product.html` 已有 `<a class="query">产品咨询</a>`，留着就行。

### 3) 基础 SEO 与适配
- URL 优化：PbootCMS 后台「全局配置 → URL 模式」选「伪静态模式」，配合 `.htaccess`（仓库已有）即可生成 `/about/24.html` 这种干净路径。
- TKD：每个栏目和每条内容后台都有独立 title/keywords/description 字段，模板里已经用 `{sort:title}` `{content:keywords}` 调好了，不用改代码。
- 图片压缩：建议接入 `tinypng-cli` 离线批处理一遍 `static/upload/`，或上传前用 squoosh，不要在线压缩（PbootCMS 没自带）。
- 海外速度：方案里没说，但要做出口品牌站，建议加 Cloudflare（免费版即可）+ 把 `skin/js/*.min.js` 走 jsDelivr CDN，海外打开速度差距很大。
- 移动适配：`media.css` 现有 36KB 移动样式，改造时**必须同步重写**，否则手机端会跟新视觉冲突。

## 六、风险与建议

1. **静态询盘表单提交至客户邮箱** —— PbootCMS 自带表单是写库的，要发邮件需要后台「全局配置 → 邮件配置」填 SMTP，**客户得提供企业邮箱 SMTP 账号**（推荐 Gmail Workspace / 阿里云企业邮）。需要提前确认。
2. **Google Maps 嵌入** —— 国内服务器访问 maps.google.com 会被墙，正式上线前确认服务器是不是放海外（出口站基本都放香港/新加坡）。
3. **多语言** —— 客户方案没明说要中英双语，但 Vargov / Lasvit 都是英文站为主。如果客户是做出口，强烈建议**纯英文站 + 可选中文**，PbootCMS 多语言要装额外插件，建议直接做英文优先。**这点要跟客户确认。**
4. **视频上传不剪辑** —— 客户要的是「上传」，但首页 hero video 文件超过 5MB 海外打开会卡，建议交付时给一份压制规范（H.264 / 8s 内 / 1080p / <3MB），客户上传前自己压。
5. **资质证书 CE/UL/ROHS** —— 是图片墙不是 PDF 下载，注意跟客户对齐展示形式。
6. **现有 wow.js + countup + animate.css** —— 老模板的滚动入场动效在新视觉里继续可用，不必删，但**幅度要收**（300ms 内、不要 fadeInLeft 这种大位移），否则跟「高端静谧」气质冲突。
